1.
先部署 (Deployment)：
將目前的程式碼上傳到 GitHub。
連結 Vercel 進行部署。
獲得： https://n8n-sheet-guard.vercel.app (假設網址)。
2.
設定 LIFF (Configuration)：
去 LINE Developers Console，把上面的網址填入 Endpoint URL。
獲得： 正式的 LIFF ID (例如 1657xxx-xxxxx)。
3.
回頭做後端整合 (Integration)：
把 LIFF ID 填入前端環境變數 (.env)。
這時候前端就能拿到 真的 User ID。
我們再撰寫 axios 呼叫 n8n API，這樣測試的資料就是真實的了。
